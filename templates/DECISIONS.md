## How to Use This File

The Development Director adds entries chronologically. Each entry documents a
technical decision, code review, guideline, or proposed change to the architecture.

### Entry Format

```markdown

**Context:** [What was the question or situation]
**Decision:** [What was decided]
**Rationale:** [Why — keep it concise]
```

### Entry Types

| Type | When to Use |
|------|------------|
| **REVIEW** | After reviewing a Development Team report |
| **DECISION** | Technical choice (library, pattern, approach) |
| **GUIDELINE** | New coding standard or convention |
| **VISION_CHANGE_PROPOSAL** | Proposing changes to ARCHITECTURE_VISION.md |
