## Status Definitions

| Status | Meaning |
|--------|---------|
| **COMPLETED** | All acceptance criteria met, all tests passing |
| **PARTIAL** | Some criteria met, work can continue in next session |
| **BLOCKED** | Cannot proceed without a decision or external input |

## Review Verdicts (set by Development Director)

| Verdict | Meaning |
|---------|---------|
| **ACCEPTED** | Work meets all criteria. Phase can close. |
| **NEEDS_FIXES** | Specific issues identified. Team must address and resubmit. |
| **REJECTED** | Fundamental problems. Rethink approach. |

## Naming Convention

```
REPORT_YYYY-MM-DD_PHASE-N.md
REPORT_YYYY-MM-DD_PHASE-N_vM.md    (revision after NEEDS_FIXES)
```

Examples:
- `REPORT_2026-03-01_PHASE-1.md`
- `REPORT_2026-03-01_PHASE-1_v2.md`
- `REPORT_2026-03-03_PHASE-2.md`
